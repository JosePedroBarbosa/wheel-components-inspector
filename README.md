# Wheel Components Inspector

An automated solution for detecting **wheels**, **rims**, and **wheel bolts** in images using YOLOv8 and computer vision. This project was developed as part of the Practical Assignment for the Artificial Intelligence course in the Informatics Engineering degree at the School of Technology and Management (ESTG), Polytechnic Institute of Porto, during the 2025/2026 academic year.

## Use Case

Automated visual inspection of wheel components in industrial and automotive workshop environments. The system detects and locates three object classes:

| Class      | Description                                    |
| ---------- | ---------------------------------------------- |
| `roda`     | Complete tire and rim assembly                 |
| `jante`    | Central metallic rim component                 |
| `parafuso` | Wheel bolts used to secure the rim to the axle |

## Installation

### Requirements

* Python 3.10 or later
* pip

### Setup

```bash
# 1. Clone the repository
git clone <repository-url>
cd wheel-components-inspector

# 2. Create a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate    # Linux/macOS
.venv\Scripts\activate       # Windows

# 3. Install the dependencies
pip install -r app/requirements.txt

# 4. Configure the Roboflow API key
cp .env_template .env
# Edit the .env file and add your API key
```

## Workflow

### 1. Dataset Download and Model Training

```bash
cd src

# Download the dataset only
python train.py \
  --workspace "jose-barbosa-rdg0j" \
  --project "wheel-components-inspector" \
  --version 6 \
  --download-only

# Run the complete training process
python train.py \
  --workspace "jose-barbosa-rdg0j" \
  --project "wheel-components-inspector" \
  --version 6 \
  --epochs 100 \
  --batch 8 \
  --device 0
```

Available arguments: `--epochs`, `--imgsz`, `--batch`, `--device` (`cpu`, `0`, or `mps`), `--run-name`, and `--download-only`.

### 2. Monitoring with TensorBoard

```bash
# Model 1
tensorboard --logdir src/runs1/train

# Model 2
tensorboard --logdir src/runs2/train

# Open http://localhost:6006
```

### 3. Preserving Model Artifacts

After training, copy the best-performing weights to the corresponding model directory:

```bash
# Model 1
cp src/runs1/train/wheel-inspector-model/weights/best.pt modelos/modelo1/weights.pt

# Model 2
cp src/runs2/train/wheel-inspector-model/weights/best.pt modelos/modelo2/weights.pt
```

### 4. CLI Inference

```bash
cd src

# Using Model 1
python infer.py \
  --model ../modelos/modelo1/weights.pt \
  --image ../test-image.jpg

# Using Model 2
python infer.py \
  --model ../modelos/modelo2/weights.pt \
  --image ../test-image.jpg
```

The script generates a JSON file containing the detections and an annotated image in the `output/` directory.

### 5. Demonstration Application

```bash
cd app
python -m streamlit run app.py
```

Features:

* Image upload for visual inspection
* Model selection and confidence threshold adjustment
* Side-by-side comparison of the original and annotated images
* Detection table and JSON output
* Result export
* Side-by-side model comparison
* Inference history

## Models

Two models were trained using the YOLOv8m architecture with weights pre-trained on the COCO dataset:

| Metric       | Model 1 (v1.0) | Model 2 (v2.0) |
| ------------ | -------------: | -------------: |
| mAP@0.5      |         0.8564 |         0.8878 |
| mAP@0.5:0.95 |         0.7116 |         0.7207 |
| Precision    |         0.8616 |         0.9113 |
| Recall       |         0.8939 |         0.9041 |

Model 2 was trained with additional high-quality images focused on wheel bolts. This resulted in a significant improvement in the detection of this class, with precision increasing from `0.64` to `0.76` and recall increasing from `0.68` to `0.71`.

Full model details:

## Dataset

* **Source:** 200 images collected by the project team and annotated using Roboflow
* **Format:** YOLOv8 with normalized bounding boxes
* **Classes:** 3 (`jante`, `parafuso`, and `roda`)
* **Base split:** 70% training, 20% validation, and 10% testing
* **Effective split:** 87.5% training, 8.3% validation, and 4.2% testing after applying Roboflow augmentation to the training set, resulting in a total of 480 images

For additional information, see the [dataset documentation](dataset/dataset.md).
